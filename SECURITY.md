# Security Policy

## Supported Versions

Security fixes are applied to the current `main` branch and the latest production deployment derived from it.
Older snapshots and ad-hoc forks are not supported unless they have been explicitly backported by the maintainers.

## Reporting a Vulnerability

Please use GitHub private vulnerability reporting for this repository whenever possible:

1. Open the repository `Security` tab.
2. Choose private vulnerability reporting / GitHub Security Advisories.
3. Include enough detail for us to reproduce and validate the issue.

If private reporting is not available in your GitHub view, do **not** open a public issue with exploit details.
Instead, contact the repository maintainers through an existing private deployment/support channel and request a confidential security handoff.

## What to Include

Please include:

- A clear description of the issue and affected area
- Steps to reproduce or a proof of concept
- Expected impact, prerequisites, and whether authentication is required
- Any suggested mitigations or patches if you already have them

## Response Expectations

The maintainers aim to:

- Acknowledge valid reports within 3 business days
- Provide a status update within 7 business days
- Coordinate a fix and disclosure timeline before any public announcement

## Disclosure Guidance

Please avoid public disclosure until the maintainers confirm that a fix or mitigation is available for affected deployments.
